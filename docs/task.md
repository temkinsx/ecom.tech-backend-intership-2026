# Тестовое задание

## Задача

Создайте HTTP-сервер на Go, который будет обрабатывать следующие эндпоинты:

- `POST /todos` — создать новую задачу  
- `GET /todos` — получить список всех задач  
- `GET /todos/{id}` — получить задачу по идентификатору  
- `PUT /todos/{id}` — обновить задачу по идентификатору  
- `DELETE /todos/{id}` — удалить задачу по идентификатору  

## Структура задачи

Задача должна включать:

- числовой идентификатор  
- заголовок  
- описание  
- признак завершённости  

## Хранилище данных

Требуется реализовать работу с данными **без использования внешних хранилищ данных**.

## Валидация

При создании и обновлении задачи **заголовок не должен быть пустым**.  
Если валидация не прошла — вернуть статус **400 Bad Request**.

## Обработка ошибок

Если задача с указанным идентификатором не найдена — вернуть **404 Not Found**.

## Тесты

Напишите **unit-тесты**.  
Убедитесь, что обрабатываются основные кейсы:

- успешное создание  
- ошибка валидации  
- дублирование идентификатора  

## Дополнительно (по желанию)

- логирование запросов  
- использование `context` для таймаутов/отмены  
- `Dockerfile` для запуска приложения  


В коде программы **не должны использоваться сторонние библиотеки**  
(допустимо использование только стандартной библиотеки).

## Результат

Прислать ссылку на репозиторий на **GitHub/GitLab**.  
В `README.md` указать, **как запустить приложение**.

## Что будет оцениваться

- качество кода (читаемость, идиоматичность Go, организация проекта)  
- обработка ошибок и крайних случаев  
- наличие и качество тестов  
- следование требованиям  
- дополнительные фичи (если реализованы уместно)
