# Todo HTTP API

Простой HTTP-сервис для управления списком задач (TODO), написанный на Go.

Используются только стандартные библиотеки и in-memory хранилище (без использования внешних БД).

## Возможности

- Создание задачи
- Получение списка всех задач
- Получение задачи по идентификатору
- Обновление задачи
- Удаление задачи
- Валидация входных данных
- Unit-тесты для репозитория и сервиса
- Логирование запросов

## Запуск локально

Убедитесь, что у вас установлен Go (версия 1.25 или выше).

В корне проекта: 
```bash
go run ./cmd/app
```

После запуска сервер будет доступен по адресу:
```
http://localhost:8080
```

## Запуск с помощью Docker

### Сборка образа
```bash
docker build -t todo-api .
```

### Запуск контейнера
```bash
docker run -p 8080:8080 todo-api
```

После этого API будет доступно по адресу:
```
http://localhost:8080
```

## API

Описание HTTP API и контрактов находится в OpenAPI-спецификации:

```
docs/openapi.yaml
```

## Тесты

Для запуска всех unit-тестов выполните:

```bash
go test ./...
```

## Примечания

- Контекст прокидывается из слоя транспорта в слой сервиса (глубже, по моему мнению, не было смысла, так как в слое репозитория не используется внешняя БД)
- Архитектура приближена к чистой (Clean Architecture)
